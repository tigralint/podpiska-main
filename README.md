## 🏗 Ключевая архитектура

Структура проекта организована для максимальной масштабируемости:

```text
├── .github/        # Стандарты сообщества (CoC, Security, Contributing)
├── views/          # Страницы (Dashboard, SubscriptionFlow, CourseFlow, и др.)
├── components/ui/  # Переиспользуемые элементы (ToneToggle, FeatureCard, ToastContainer)
├── hooks/          # Бизнес-логика (useClaimForm, useClaimFlow, useFocusTrap)
├── services/       # API-клиенты (geminiService, fetchWithRetry)
├── api/            # Серверные функции (generateClaim, rateLimiter, promptBuilder)
├── store/          # Глобальное состояние (Zustand)
├── utils/          # Хелперы (clipboard, errors, format)
└── types/          # Строгие TypeScript интерфейсы
```

### Интересные инженерные решения
* **`useClaimForm<T>`** — Универсальный generic-хук с `AbortController`, мощной валидацией и механизмом retry. Обслуживает все флоу генерации.
* **Smart Prompt Builder** — Конструктор промптов вынесен в отдельный модуль (`promptBuilder.ts`), изолируя логику составления текстовых шаблонов от API-хэндлеров.
* **RateLimiter** — Написан отдельным классом с внедряемой зависимостью времени (`injectable now`) для 100% тестируемости.
* **Discriminated Unions** — TypeScript строго следит за структурой (например, поле `calculatedRefund` доступно только в контексте курсов).

## 🛡 Безопасность и надёжность

Мы серьёзно относимся к защите API и пользовательских данных:
- **Cloudflare Turnstile**: Серверная невидимая верификация (защита ИИ от ботов).
- **Rate Limit**: Жёсткие лимиты (60 запросов/час на IP) с автоматической блокировкой запросов (Fail-Closed) при падении базы данных для защиты бюджета.
- **Runtime Type Guards**: Строгая валидация тела запроса (Zod `.max()`) перед обращением к нейросети, защищающая память от переполнения гигантскими Payload-ами.
- **Input Sanitization**: Удаление опасных символов (`<>{}[]`) из ввода пользователя для защиты LLM-промптов.
- **Глубинная защита (Defense-in-Depth)**: Строгий `Content-Security-Policy` и `CORS`, блокирующие исполнение сторонних скриптов.
- **Graceful Degradation**: Обширное использование `ErrorBoundary`, кастомные fallback-интерфейсы.

## 🚀 Как запустить локально

1. **Клонируйте репозиторий и установите зависимости:**
   ```bash
   git clone https://github.com/tigralint/chestnayapodpiska.git
   cd podpiska
   npm install
   ```

2. **Настройте переменные окружения:**
   Создайте файл `.env` в корне (за основу можно взять `.env.example`):
   ```env
   OPENROUTER_API_KEY=sk-or-...
   VITE_TURNSTILE_SITE_KEY=0x...
   TURNSTILE_SECRET_KEY=0x...
   UPSTASH_REDIS_REST_URL=https://...
   UPSTASH_REDIS_REST_TOKEN=...
   ```

3. **Запустите проект:**
   ```bash
   # Для фронтенда
   npm run dev        # Доступно на localhost:3000
   
   # Для полноценной локальной симуляции Vercel API-функций:
   npx vercel dev
   ```

4. **Прогон тестов (124 test cases, 100% pass rate):**
   ```bash
   npm test
   ```
